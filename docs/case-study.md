# Case Study: Generative Design Decision Explorer

## Type

Self-directed concept project

## Problem

Generative design tools create many possible engineering options. Those options still need to be organized, compared, explained, and reviewed before a team makes a decision.

## Goal

Build a mock decision dashboard that helps a technical stakeholder compare generated motorsport part options using clear data, constraints, and tradeoffs.

## Scenario

A fictional motorsport team is reviewing options for a front suspension mounting bracket.

## Inputs

- Preserve geometry
- Obstacle geometry
- Loads and forces
- Materials
- Manufacturing methods
- Decision rules

## Outputs Compared

- Weight
- Factor of safety
- Cost level
- Lead time
- Risk notes
- Recommendation

## Process

1. Defined the part scenario.
2. Created mock constraint data.
3. Created mock design option data.
4. Built a dashboard interface in React and Tailwind CSS.
5. Added a comparison table and chart.
6. Added a recommendation panel.
7. Documented the limits of the concept.

## Outcome

The project demonstrates how Information Science supports AI-assisted engineering workflows by turning many generated options into clear, reviewable decisions.

## Key Learning

The value of generative design is not only the shape it creates. The value also comes from how teams understand the options, compare tradeoffs, and decide what deserves deeper review.
